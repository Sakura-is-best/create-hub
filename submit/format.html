格式规范

一、文章类型

本刊接受以下两类稿件：

类型说明建议字数
研究论文针对某个机制或工程进行系统性的测试、分析与优化2000～4000 字
技术简报分享一个小技巧、简易装置或实用蓝图800～1500 字

二、论文结构

每篇研究论文必须包含以下部分（技术简报可适当简化）：

  1. 标题：简明扼要，一般不超过 25 字。
  2. 作者：游戏 ID 或 GitHub 用户名。
  3. 摘要：200 字以内，概括研究背景、方法、结果与结论。
  4. 引言：研究动机、现有方案不足、创新点。
  5. 核心原理：涉及的机械动力机制。
  6. 实验 / 设计：建造步骤、测试环境、数据表格。
  7. 结论：总结与展望。
  8. 参考文献（可选）。

三、元数据（Front Matter）

每个 Markdown 文件开头必须包含 YAML 格式的 Front Matter：

--- title: "基于机械动力的树场自动化构建及其效能分析" authors: ["创世研究员", "红石博士"] date: 2026-06-19 summary: "采用递归种植-定向采伐策略，原木产量提升约40%。" tags: ["树场", "自动化", "动力传输"] ---

四、审稿标准

标准说明
创新性是否提出了新的设计方案或优化思路
实用性设计能否在生存模式中正常建造和使用
可读性行文流畅，结构清晰，图文并茂
数据支撑结论是否有测试数据或对比验证

五、投稿方式

请将写好的 Markdown 文件放入 content/publications/ 目录，通过 Pull Request 提交。
详细流程请参阅 投稿指南。
